# ToDo-API-BSBO

## Описание проекта
API для управления задачами с использованием матрицы Эйзенхауэра. Позволяет создавать, просматривать, фильтровать и искать задачи по различным критериям.

## Технологии
- **FastAPI** - веб-фреймворк для создания API
- **Python 3.13+** - язык программирования
- **Uvicorn** - ASGI сервер для запуска приложения
- **Pydantic** - валидация данных
- **APIRouter** - модульная маршрутизация (Часть 2)

## Матрица Эйзенхауэра
- **Q1**: Важно + Срочно (Кризисы, дедлайны)
- **Q2**: Важно + Не срочно (Планирование, развитие)
- **Q3**: Не важно + Срочно (Отвлечения, рутина)
- **Q4**: Не важно + Не срочно ("Воры" времени)

## Архитектура проекта

### Часть 1: Базовое API (FastAPI)
Простая монолитная структура с одним файлом `main.py`.

### Часть 2: Модульная архитектура (APIRouter)
Рефакторинг проекта с использованием `APIRouter` для улучшения структуры:

#### Структура проекта:
ToDo-API-BSBO/
├── routers/ # Модули маршрутизации
│ ├── init.py # Инициализация пакета
│ ├── tasks.py # Все endpoints для работы с задачами
│ └── stats.py # Endpoint для получения статистики
├── main.py # Точка входа приложения
├── models.py # Pydantic модели
├── schemas.py # Схемы данных
├── database.py # Работа с базой данных
├── requirements.txt # Зависимости Python
├── README.md # Документация
└── .gitignore # Игнорируемые файлы Git


#### Преимущества новой архитектуры:
- ✅ **Модульность** - каждый файл отвечает за свою логику
- ✅ **Читаемость** - код организован логически
- ✅ **Масштабируемость** - легко добавлять новые модули
- ✅ **Группировка в документации** - эндпоинты сгруппированы по тегам в Swagger UI

## Доступные эндпоинты

### Основные
- `GET /` - Информация об API
- `GET /tasks` - Все задачи
- `GET /tasks/stats` - Статистика по задачам

### Фильтрация
- `GET /tasks/{task_id}` - Задача по ID
- `GET /tasks/quadrant/{quadrant}` - Задачи по квадранту (Q1-Q4)
- `GET /tasks/status/{status}` - Задачи по статусу (completed/pending)

### Поиск
- `GET /tasks/search?q={query}` - Поиск по названию/описанию
